# Turret Calibration System Updates

This update fixes and enhances the turret calibration system. The Elastic dashboard layout (`elastic/turret-calibration-layout.json`) was completely rewritten to use the correct schema format—including proper root structure with `tabs` array, pixel-based positioning, ARGB decimal color values, and correctly named widget types like `"Boolean Box"` and `"Text Display"`. An alliance selection feature was added across the calibration pipeline: `CalibrationDataRecorder` now includes alliance in its data model and CSV output, `TurretCalibrationCommand` publishes alliance options via NetworkTables for a ComboBox Chooser widget, and `GenerateLookupTables` parses the alliance column (with backward compatibility) and includes alliance metadata in generated lookup table headers. The calibration data is distance-based, making it inherently alliance-agnostic since the same hood/flywheel settings apply regardless of which alliance the robot is on.

Additionally, two runtime issues were resolved. The `DriverStation.isTest()` check in `RobotContainer` was moved from constructor-time (where it always returned false due to the driver station not being connected yet) to runtime using `RobotModeTriggers.test()`, ensuring test mode bindings activate properly when test mode is selected. PhotonVision's "coprocessor is not sending new data" warnings during simulation were eliminated by calling `PhotonCamera.setVersionCheckEnabled(false)` when running in simulation mode.
